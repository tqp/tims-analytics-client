Authentication

Authentication is performed through a custom developed Spring security implementation using JSON Web Tokens (JWT). Usernames and passwords are sent securely to the server via a REST call. The data provided is validated against a mySQL database. A token is generated and passed back to the client. This token is then used to perform any subsequent calls to the server. Spring blocks any calls to the server that do not include a valid and authenticated token.

Data Source

This app uses an AWS-based mySQL database for authentication and other data-related pages through a basic JDBC connection.

Hosting

The client side of this app is hosted on AWS through the Amplify Service. Amplify is very convenient to work with because AWS will automatically build and deploy the front end any time changes are committed to GitHub.
The server side of this app is hosted on AWS through the Elastic Beanstalk service.
